# PSAddonBYFlow

Дополнение к плагину [ProtectionStones](https://github.com/espidev/ProtectionStones) для Paper/Spigot 1.21.1.

## Возможности
- Каждому приватному региону назначается индивидуальный запас "жизней" (по умолчанию 3), который сохраняется в `plugins/PSAddonBYFlow/data.yml`.
- Для каждого типа блоков ProtectionStones можно задать собственные значения жизней, урона от взрыва, текста и поведения голограммы.
- Урон списывается при каждом взрыве (по умолчанию только от TNT), а текущее значение отображается в голограмме над блоком привата.
- При обнулении жизней регион автоматически удаляется через API ProtectionStones.
- Встроенная защита от бага установки приватных блоков вплотную друг к другу (стэкинг). При попытке создать регион поверх уже существующего событие отменяется, а игрок получает сообщение.
- Взрывы визера, Wither Skull и любые попытки сломать приватный блок визером больше не обходят систему жизней.
- Поддерживается собственный движок голограмм на TextDisplay и автоматическое подключение к [DecentHolograms](https://www.spigotmc.org/resources/decent-holograms.96927/) при наличии плагина.
- Голограммы поддерживают HEX-цвета, многострочный текст и плейсхолдеры {owner}, {world}, координаты и другие значения.

## Установка
1. Соберите плагин при помощи установленного Gradle 8+ командой `gradle build` (требуется JDK 21, как и на сервере Paper 1.21.1). При необходимости можно сгенерировать локальный wrapper (`gradle wrapper`) — бинарный `gradle-wrapper.jar` намеренно не хранится в репозитории.
2. Положите скомпилированный `.jar` файл в папку `plugins` вместе с установленным ProtectionStones.
3. Перезапустите сервер. Плагин создаст конфигурацию и файл `data.yml` для хранения жизней регионов.

## Конфигурация
`config.yml` поддерживает значения по умолчанию и точечные переопределения для разных материалов:

```yaml
default:
  lives: 3
  damage-per-explosion: 1
  tnt-only: true
  hologram:
    enabled: true
    offset-y: 1.8
    lines:
      - "&cЖизни привата: &#ff5555{lives}&7/&f{max}"
      - "&7Владелец: &f{owner}"

blocks:
  DIAMOND_BLOCK:
    lives: 5
    hologram:
      lines:
        - "&bАлмазный приват"
        - "&f{lives}&7/&f{max} &8жизней"

prevent-stacking: true

messages:
  stack-block-denied: "&cНельзя ставить приват вплотную к другому приватному блоку!"
```

- Раздел `default` применяется ко всем приватным блокам, для которых нет переопределений.
- В `blocks` материал указывается так же, как в Bukkit API (`DIAMOND_BLOCK`, `EMERALD_BLOCK`, ...). Можно задать любое количество блоков.
- Параметр `tnt-only` в каждом блоке определяет, нужно ли реагировать только на TNT или на любые взрывы. Даже при `tnt-only: true` приватный блок защищён от визера.
- Флаг `prevent-stacking` включает/отключает проверку на наложение приватных блоков друг на друга.
- Сообщение `stack-block-denied` отправляется игроку при попытке поставить приват вплотную к другому.
- Для голограмм доступны плейсхолдеры `{lives}`, `{max}`, `{owner}`, `{owner_uuid}`, `{region}`, `{world}`, `{x}`, `{y}`, `{z}`, `{material}`. Можно использовать HEX цвета (`&#ff0000` или `<#ff0000>`) и многострочный формат через список строк.

После изменения конфигурации перезапустите сервер (или выполните полную перезагрузку плагинов), чтобы настройки применились.
